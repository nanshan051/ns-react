    ES6 模块化规范

    ES6exportimport.mjs

    ES6 模块化规范是一个 官方标准 的规范，它是在语言标准的层面上实现了模块化功能，是目前 最流行的 模块化规范，且浏览器与服务器均支持该规范。

    # 1. 初步体验

    # 1.1. school.js

    // 通过在变量、函数前添加export的⽅式，分别导出数据
export const name = "北京大学";
export const predecessor = "京师大学堂";
export function getTel() {
  return "010-62751407";
}
function getCities() {
  return ["北京", "无锡", "深圳"];
}

    # 2. 浏览器端运行

    在浏览器运行 ES6 模块时，需要对 <script> 标签设置 type="module" 。

    <script type="module" src="./index.js"></script>
    1

    # 3. Node.js 中运行

    如果直接通过右键『Run Code』运行 ES6 模块，会报错：

    解决方案：

    • 方式一：在 package.json 中添加 "type": "module"。
    • 方式二：将 JavaScript 文件后缀从 .js 改为 .mjs ，Node 则会自动识别 ES6 模块。

    这里以方式二为例，添加 package.json 文件：

    {
  "type": "module"
}

    右键 index.js ，选择 『Run Code』 ，该 ES6 模块在 Node.js 环境下成功运行。

    # 4. 导出数据

    ES6 模块化提供 3 种导出方式：① 分别导出； ② 统一导出； ③ 默认导出。

    # 5. 导入数据

    对于 ES6 模块化来说，使用何种 导入方式，要根据 导出方式 决定。

    # 5.1. 导入全部 通用

    可以将模块中的所有导出内容整合到一个对象中。

    import * as school from "./school.js";
    1

    # 5.2. 命名导入 分别导出 统一导出 📌

    导出数据的模块：

    // 分别导出

export const predecessor = "京师大学堂";
function getTel() {
  return "010-62751407";
}
function getCities() {
  return ["北京", "无锡", "深圳"];
}

// 统一导出

    注意：命名导入中的 {}

    • 所以里面的变量 不是 对象解构赋值！！！

    # 6. 数据引用问题 📌

    • 思考 1：如下代码的输出结果是什么？（不要想太多，不涉及模块化相关知识）
    function count() {
  let sum = 1;
  function increment() {
    sum += 1;
    console.log(sum);
  }
  return { sum, increment };
}
const { sum, increment } = count();
console.log(sum);
increment();
increment();
console.log(sum);

    count 函数存在闭包，每次 increment() 执行后，count 函数内的 sum 都会发生变化。但是为什么外部 sum 一直都是 1 呢？

    这是因为，count 外部 sum 是通过『对象解构赋值』重新定义的一个变量，与 count 内部 sum 无关。
